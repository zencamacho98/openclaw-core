# Milestone Map — The Abode
*Direction markers, not deadlines. Reality check against doctrine.*

---

## Near term — Stable workforce foundation

- Peter is a strong, reliable front door. Responses are useful, not just routed.
- Neighborhood UI reflects real work and real state. No theater substituting for outcomes.
- Belfort demonstrates serious paper-trading readiness: risk controls, readiness scorecard consistently passing, measurable and explainable performance, realistic trade cost modeling.
- Market/news awareness: **does not currently exist at all** — building this is a near-term gap, not an incremental improvement.
- Belfort's live trading readiness is honestly assessed — not declared by schedule.
- Frank Lloyd: **docs-level definition done** (role, eligibility criteria, boundary in ROLE_MAP). Build milestone is separate and not yet started — "defined" here means a clear spec exists, not that it is built.
- Docs reflect actual doctrine and architecture (this vision reset is the start of that).
- Reliability improves: fewer crashes, less manual intervention, fewer babysitting cycles.
- Token waste decreases: cheap-tier handles routine work, strong-tier used sparingly.

---

## Mid term — First economic value proof

- At least one house creates real economic value or clearly covers its operating costs.
- Peter handles more coordination with less operator hand-holding.
- Reusable platform components (learning infrastructure, readiness framework, observability) are clearly separated from house-specific logic.
- Autonomy escalation: at least one house can make routine decisions without operator confirmation.
- Frank Lloyd can accept a spec and execute a build autonomously.

---

## Long term — Autonomous business ecosystem

- Multiple houses create net value.
- Proven houses can be duplicated or productized for other contexts.
- Parts of the platform or individual houses can be sold separately.
- The Abode requires operator attention only for strategic decisions, not routine operation.
- Earned autonomy is demonstrably higher: the system has proven it can be trusted with larger scope.

---

## What is not on the map

- Discord integration — not a current priority
- High-frequency trading — not the target
- Public multi-tenant product — not yet defined
- Full AGI autonomy theater — explicitly ruled out

---

## Autonomy doctrine

Autonomy escalates only when:
1. Evidence shows the agent performs reliably in the relevant domain
2. Safeguards exist for the failure modes that matter
3. The operator has reviewed and accepted the autonomy grant

No house gets autonomous authority by declaration or by default.
