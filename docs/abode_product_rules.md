# Abode product rules

## Product direction
The Abode should be understandable for a normal user, not only for a developer.

The main user-facing feeling should be:
- I talk to Peter
- I watch Belfort trade and learn
- I can understand what is happening quickly
- I can act on important decisions without digging through technical clutter

## Neighborhood rule
The neighborhood is the main user-facing front layer.

The dev/dashboard UI remains:
- fallback
- advanced area
- deeper control surface

## Neighborhood priorities
- plain English over internal jargon
- strong selected focus
- glanceable status
- compact summary
- calm, readable interaction
- core user loop should happen in the neighborhood when practical

## Belfort presentation rules
Use trading-first language:
- Trading ON/OFF
- Research ON/OFF (not "loop on/off")
- current position, last trade, cash, P&L, trade count
- avoid surfacing internal campaign IDs prominently in the neighborhood

## Peter presentation rules
Peter should feel like:
- the front door
- the guide
- the coordinator
- the one who tells the user what matters now

Peter should not feel like:
- a raw command router
- Belfort
- a checker layer
- a developer console
