# Abode Delivery Report

Use this skill at the end of each implementation block.

## Report format
Always return:

### What changed
A concise summary of the completed work.

### Files added/edited
List exact files touched.

### Reused flows
Call out existing backend endpoints, UI panels, or control flows that were reused.

### Scope control
State what was deliberately left out to keep the block lean.

### Remaining gaps
List only the most meaningful remaining gaps.

### Next block
Recommend one exact next block, not multiple competing directions.

## Style rules
- be practical
- be specific
- avoid vague "could also do X/Y/Z"
- prefer one recommended next step
- call out scope drift if it happened
